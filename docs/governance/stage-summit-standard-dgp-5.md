# Stage Summit Standard — DGP-5

Status: **CURRENT** | Date: 2026-05-09 | Phase: DGP-5
Authority: current_status | Owner: Governance

## Purpose

A stage summit is a compressed summary of a group of completed phases, designed for human and AI onboarding into the current state. It is context-compression evidence, NOT permanent truth by default.

## Required Fields

1. **Stage scope** — What set of phases or domain this summit covers.

2. **Phase list** — Which phases are included, each with status (CLOSED/PARTIAL/BLOCKED).

3. **What changed** — Concrete changes, not "improved governance."

4. **Current state** — What the system looks like NOW after these phases.

5. **Evidence summary** — Key metrics: BLOCKED count, DEGRADED count, test count, reconciler output.

6. **Remaining debts** — What is still open and why.

7. **What not to infer** — Explicit boundaries on what this summit does NOT mean.

8. **Next route** — What phase(s) come next.

9. **New AI reading update** — What a new AI should read to understand this state.

10. **Non-authorization boundary** — Must state: this summit does not authorize merge/release/deploy.

## Hard Rules

- Stage summit is context compression, not permanent truth. Register in current-truth-entry-map if it carries truth claims.
- Stage summit must NOT say "全线闭合" unless every phase in scope is CLOSED with verified evidence.
- What not to infer is NOT optional — it prevents downstream over-reading.
- Stage summit is supporting_evidence by default.
