# External Benchmark Reading Guide

> For AI agents reading Ordivon governance documentation.

## Purpose

EGB-1 provides external AI governance benchmarks for internal comparison.
This guide explains how to read and use them safely.

## What EGB-1 Is

A reference layer that maps Ordivon concepts to five external governance
frameworks: OpenAI Preparedness Framework, Anthropic Responsible Scaling
Policy, Google DeepMind Frontier Safety Framework, NIST AI RMF, and
ISO/IEC 42001.

## What EGB-1 Is Not

EGB-1 is **not**:
- A compliance certification
- An endorsement of any external framework
- A claim of equivalence
- A binding policy document
- Legal advice
- A public standard

## Safe Reading Rules

When reading EGB-1 documents:

1. **"Maps to" does not mean "equivalent to."** The mapping shows conceptual
   parallels, not identical implementations.

2. **"Gap" does not mean "defect."** Gaps are observations about where
   Ordivon differs from external benchmarks. Some gaps are intentional
   (Ordivon-native innovations without external precedent).

3. **External frameworks are benchmarks, not authorities.** Ordivon does
   not defer to external frameworks for governance decisions.

4. **EGB-1 is reference-only.** It cannot be used to authorize action,
   claim compliance, or override Ordivon governance.

## Key Ordivon Distinctions

Ordivon has concepts that no external framework addresses:

- **Closure Predicate**: phase can only close when all conditions met
- **NO-GO Ladder**: permanently prohibited actions
- **CandidateRule → Policy**: advisory observation before binding enforcement
- **New AI Context Check**: governance docs verified for AI readability
- **Receipt/Debt/Gate triad**: every action has all three

When comparing Ordivon to external benchmarks, remember that Ordivon is
a **governance operating system**, not a safety policy document. External
frameworks define what should be done; Ordivon defines how to verify it
was done correctly.
